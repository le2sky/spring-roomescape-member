## 구현 기능 목록

### 1단계

- [ ] 예외 처리
    - [x] 시간 생성 시 시작 시간에 유효하지 않은 값이 입력되었을 때
    - [x] 예약 생성 시 예약자명, 날짜, 시간에 유효하지 않은 값이 입력 되었을 때
    - [ ] 특정 시간에 대한 예약이 존재하는데, 그 시간을 삭제하려 할 때
    - [ ] 사용자로부터 잘못된 요청이 오면 BAD_REQUEST 를 응답한다
- [ ] 추가된 서비스 정책
    - [ ] 중복 예약은 불가능하다
    - [ ] 지나간 날짜와 시간에 대한 예약은 생성 불가능하다
